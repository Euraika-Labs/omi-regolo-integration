### 1. Provider-aware index selector in `backend/database/vector_db.py`

Add at the top of the file:

```python
def _get_index_for_provider(provider: str | None):
    """Return the Pinecone index handle for a given embedding provider.

    Provider 'regolo' (4096-dim Qwen3) routes to PINECONE_INDEX_NAME_EU.
    Anything else (legacy / OpenAI / Gemini, all 3072-dim) routes to the
    canonical PINECONE_INDEX_NAME. Returns None if the requested provider's
    index isn't configured — caller must handle gracefully.
    """
    if provider == 'regolo':
        name = os.getenv('PINECONE_INDEX_NAME_EU', '').strip()
        return pc.Index(name) if name else None
    name = os.getenv('PINECONE_INDEX_NAME', '').strip()
    return pc.Index(name) if name else None
```

Then refactor every existing `index.upsert(...)` / `index.query(...)` / `index.delete(...)` call to use `_get_index_for_provider(provider)` where `provider` is sourced from the request's privacy-mode resolution. Match the in-memory `index = pc.Index(...)` initialization at module top — keep it as the legacy default for backwards compat with existing callers.

### 2. Augment vector metadata with `(provider, model, dim)`

The current `_get_data` (line 21 of upstream `vector_db.py`) emits:

```python
{
    "id": f'{uid}-{conversation_id}',
    "values": vector,
    "metadata": { "uid": uid, "memory_id": conversation_id, "created_at": ... },
}
```

Add three keys to metadata:

```python
"metadata": {
    "uid": uid,
    "memory_id": conversation_id,
    "created_at": ...,
    "provider": provider,           # 'openai' | 'regolo' | 'gemini'
    "model": model,                 # e.g. 'text-embedding-3-large'
    "dim": dim,                     # 3072 or 4096
},
```

This is **non-breaking** — Pinecone ignores unknown metadata fields; existing readers keep working. Run a one-shot backfill (script template in §3) to tag existing 3072-dim vectors with `{provider:'openai', model:'text-embedding-3-large', dim:3072}`.

### 3. One-shot backfill script: `backend/migrations/006_byok_provider_metadata.py`

```python
"""
Migration 006: backfill (provider, model, dim) metadata on existing vectors.

Run-once. Idempotent — if 'provider' is already set we skip. Iterates each
namespace (ns1 conversations, ns2 memories, ns3 screen activity) and updates
metadata via Pinecone's index.update API. ~30K vectors at 100/sec ≈ 5 min.
"""
import os
from pinecone import Pinecone

LEGACY_TAGS = {
    'ns1': {'provider': 'openai',  'model': 'text-embedding-3-large', 'dim': 3072},
    'ns2': {'provider': 'openai',  'model': 'text-embedding-3-large', 'dim': 3072},
    'ns3': {'provider': 'gemini',  'model': 'embedding-001',          'dim': 3072},
}

pc = Pinecone(api_key=os.environ['PINECONE_API_KEY'])
index = pc.Index(os.environ['PINECONE_INDEX_NAME'])

for ns, tags in LEGACY_TAGS.items():
    cursor = None
    while True:
        page = index.list_paginated(namespace=ns, pagination_token=cursor, limit=100)
        for v in page.vectors:
            if v.metadata.get('provider'):
                continue  # idempotent
            index.update(id=v.id, namespace=ns, set_metadata=tags)
        if not page.pagination.next:
            break
        cursor = page.pagination.next
```

### 4. Update write-path call sites to consult Privacy Mode

The audit's privacy-write gap (`process_conversation.py:580`, `app_integrations.py:206`, `app_integrations.py:338`) — refactor each to:

```python
from utils.llm.eu_privacy import resolve_feature_model, FeatureRouteKind
from utils.llm.clients import generate_embedding, generate_regolo_embedding

def save_structured_vector(uid: str, conversation: Conversation, ...):
    route = resolve_feature_model(uid, 'memory_search')  # or appropriate feature
    if route.kind is FeatureRouteKind.REGOLO:
        vector = generate_regolo_embedding(content)
        upsert_vector(uid, conversation_id, vector, provider='regolo',
                      model='Qwen3-Embedding-8B', dim=4096)
    elif route.kind is FeatureRouteKind.PRIMARY:
        vector = generate_embedding(content)
        upsert_vector(uid, conversation_id, vector, provider='openai',
                      model='text-embedding-3-large', dim=3072)
    else:
        # HARD_BLOCK — should not reach here if the upstream caller checked
        # the feature's availability; raise to surface the bug.
        raise RuntimeError(f'embedding write attempted on hard-blocked feature: {route.banner}')
```

This is the actual privacy-promise fix called out in the M2 audit. Only safe to merge after the M2.5 branch (this one) lands AND the second Pinecone index is provisioned.

## Cost considerations

- Second Pinecone index: ~$0.10/hr serverless or starter pod. Deferred budget impact ≤ $80/mo at staging scale; production cost depends on vector volume, query rate, and replica config. Get finance sign-off before provisioning.
- Qwen3-Embedding-8B at €0.001/req — embedding workload is read-heavy on existing vectors, write-light, so the marginal Regolo cost is small (most queries hit cached vectors).
- Backfill migration is a one-shot cost: 30K vectors × Pinecone update API ≈ free; no embedding regeneration in this migration (we keep the existing 3072-dim OpenAI vectors as legacy and only tag their metadata).

## Mid-account toggle policy (from M2 audit § Open questions)

Recommended **Option A**: parallel histories. When a user toggles Privacy Mode ON:

- Old (3072-dim OpenAI) vectors stay queryable when they later toggle OFF — backward compat.
- New conversations from that point forward get embedded at 4096-dim Qwen3 and live in the EU index.
- Search results merge across both indexes if the user has data in both — query both, top-k merge by score. Cosine similarity is comparable across 3072/4096 within reasonable bounds; document the small-quality caveat in release notes.
- No destructive re-embed. Users can opt into a manual "re-embed everything for stronger EU compliance" later if we want a P2 enhancement.
