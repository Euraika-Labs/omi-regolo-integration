**For:** infra + backend operators provisioning the EU embedding index.
**Companion to:** `M2-embedding-audit.md` (the original HARD STOP audit).

## What ships in this repo

The patch package ships M2.5 in two MRs:

| MR | Branch | Scope |
|---|---|---|
| !6 | `m2.5-embedding-migration` | Code half — `_RegoloEmbeddingProxy` + `eu_privacy.eu_embedding_index_provisioned()` env-gate + dispatcher route lift |
| !7 | `m2.5-vector-store-patches` (depends !6) | Vector-store §1-4 — provider-aware Pinecone index selector, additive `(provider, model, dim)` metadata, backfill migration 007, privacy-mode-aware `save_structured_vector` |

## What §1-4 actually changes

### §1 Provider-aware Pinecone index selector — `backend/database/vector_db.py`

```python
_EU_INDEX = None
if pc is not None and os.getenv('PINECONE_INDEX_NAME_EU', '').strip():
    _EU_INDEX = pc.Index(os.getenv('PINECONE_INDEX_NAME_EU', '').strip())


def _get_index_for_provider(provider: str = None):
    if provider == 'regolo':
        return _EU_INDEX
    return index
```

Existing reads/writes still go through the legacy `index` handle by default. Callers that pass `provider='regolo'` get the EU 4096-dim index when provisioned, None otherwise.

### §2 Additive `(provider, model, dim)` metadata

`_get_data` now accepts kw-only `provider`, `model`, `dim` (defaults: `openai` / `text-embedding-3-large` / `3072`). Vector metadata carries those keys. Non-breaking — Pinecone ignores unknown metadata fields and existing readers don't filter on these.

### §3 One-shot backfill migration `007_byok_provider_metadata.py`

Tags every vector in the legacy 3072-dim index with the schema attribution that the new write paths now stamp on every upsert. Idempotent + supports `--dry-run`. Per-namespace tags:

| Namespace | Provider | Model | Dim |
|---|---|---|---|
| ns1 (conversations) | `openai` | `text-embedding-3-large` | 3072 |
| ns2 (memories) | `openai` | `text-embedding-3-large` | 3072 |
| ns3 (screen activity) | `gemini` | `embedding-001` | 3072 |

Estimate ~5 min for 30K vectors at 100/sec.

### §4 Privacy-mode-aware `save_structured_vector`

`backend/utils/conversations/process_conversation.py:580` now resolves `eu_privacy.resolve_feature_model(uid, 'memory_search')` and routes:

- `REGOLO` → `generate_regolo_embedding` (4096-dim Qwen3-Embedding-8B)
- `PRIMARY` → existing `generate_embedding` (3072-dim OpenAI)
- `HARD_BLOCK` → log + skip the write entirely

This closes the privacy-write gap from the M2 audit (MR !5).

**The two query-side sites** (`app_integrations.py:206` `_retrieve_contextual_memories`, `app_integrations.py:338` proactive context vector) need a paired refactor of `query_vectors_by_metadata` to consult `_get_index_for_provider`. Tracked as a follow-up; out of scope for MR !7.

## Operator deployment checklist

In order:

- [ ] **Review and merge !6** on GitLab (sister-repo M2.5 code half).
- [ ] **Review and merge !7** on GitLab (vector-store §1-4 patches; rebases cleanly on main after !6 lands).
- [ ] **Provision the second Pinecone index** at 4096-dim. Suggested name `omi-eu-prod-4096`. Match the existing index's metric type (cosine).
- [ ] **Set env vars** in the production deploy:
  - `PINECONE_INDEX_NAME_EU=omi-eu-prod-4096` — surfaces the new index to `eu_privacy.eu_embedding_index_provisioned()`.
  - `REGOLO_API_KEY=<key>` — env fallback for users without a BYOK Regolo key.
- [ ] **Run the backfill migration once** in staging, then production:
  ```bash

  python backend/migrations/007_byok_provider_metadata.py --dry-run
  # then without --dry-run when satisfied
  ```
  Watch logs for "skipped (already tagged)" — if every vector skips, nothing is wrong.
- [ ] **Lift the EU Privacy Mode "embedding-dependent features blocked" disclaimer** in the product copy (Settings card on web frontend + desktop client) — the user-visible promise becomes deliverable once the dispatcher routes embedding writes to Regolo.
- [ ] **Telemetry sanity** — Grafana `provider=regolo` histogram should show non-zero `embedding_dim=4096` vectors after the first EU-mode user processes a conversation.

## Cost considerations

- Second Pinecone index: ~$0.10/hr serverless or starter pod. Deferred budget impact ≤ $80/mo at staging scale; production cost depends on vector volume, query rate, and replica config. **Get finance sign-off before provisioning.**
- Qwen3-Embedding-8B at €0.001/req — embedding workload is read-heavy on existing vectors, write-light, so the marginal Regolo cost is small (most queries hit cached vectors).
- Backfill migration is essentially free: 30K vectors × Pinecone update API ≈ no per-call cost. No embedding regeneration in this migration — we keep the existing 3072-dim OpenAI vectors as legacy and only tag their metadata.

## Mid-account toggle policy

Recommended **Option A**: parallel histories.

When a user toggles Privacy Mode ON:

- Old (3072-dim OpenAI) vectors stay queryable when they later toggle OFF — backward compat.
- New conversations from that point forward get embedded at 4096-dim Qwen3 and live in the EU index.
- Search results merge across both indexes if the user has data in both — query both, top-k merge by score. Cosine similarity is comparable across 3072/4096 within reasonable bounds; document the small-quality caveat in release notes.
- No destructive re-embed. Users can opt into a manual "re-embed everything for stronger EU compliance" later if it becomes a P2 enhancement request.
